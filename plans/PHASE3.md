# Autonomous Agent Directive: ORAS Integration & End-to-End Validation

## 1. Objective and Context

The previous validation attempt verified the HTTP POST payload validation but failed to verify the asynchronous ORAS registry integration. A 201 Created response only means the record was written to the database. The core requirement of Phase 2 is to ensure the reconciliation loop successfully connects to an OCI registry, pulls a manifest, and extracts metadata.

You must perform a full end-to-end integration test by standing up a local OCI registry, pushing a properly formatted artifact, and verifying the reconciliation state transitions.

## 2. Environmental Prerequisites

Before modifying any Go code, you must ensure your local environment has the required testing infrastructure. You will need:

* **Docker:** To run a local OCI registry.
* **ORAS CLI:** To construct and push the test artifact.

## 3. Local Registry Staging Protocol

Execute the following commands to stage a live OCI registry on your local machine and populate it with a valid firmware bundle artifact. Do not skip this step.

1. **Start the local registry daemon:**
```bash
docker run -d -p 5000:5000 --name local-oci-registry registry:2

```


2. **Create a dummy firmware binary:**
```bash
echo "01010101" > dummy_firmware.bin

```


3. **Push the artifact to the local registry:**
You must use the exact artifact type required by the service's business logic.
```bash
oras push localhost:5000/firmware/test-bmc:1.0.0 \
  --artifact-type application/vnd.openchami.firmware.bundle.v1+json \
  --annotation "org.openchami.firmware.component=bmc" \
  --annotation "org.openchami.firmware.version=1.0.0" \
  dummy_firmware.bin:application/vnd.openchami.firmware.payload.v1

```



## 4. Execution and Verification Sequence

With the registry staged and populated, you must run the service and explicitly verify the background reconciliation loop.

1. **Start the Fabrica service:**
Run the service in the background or a separate terminal.
```bash
go run ./cmd/server serve --database-url="file:test.db?cache=shared&_fk=1"

```


2. **Create the FirmwareBundle resource:**
Point the resource to the local registry you just staged.
```bash
curl -sS -X POST http://127.0.0.1:8080/firmwarebundles/ \
  -H 'Content-Type: application/json' \
  -d '{
    "metadata": {"name": "local-test-bundle"},
    "spec": {
      "registryURL": "localhost:5000",
      "repository": "firmware/test-bmc",
      "tagOrDigest": "1.0.0"
    }
  }'

```


3. **Verify the Reconciliation State (CRITICAL STEP):**
You must query the resource to confirm the background loop successfully interacted with the ORAS client. Execute the following GET request:
```bash
curl -sS http://127.0.0.1:8080/firmwarebundles/local-test-bundle/

```


* **Acceptance Criteria:** The JSON response MUST contain `"discovered": true` within the `status` object. It must also contain the extracted annotations in `status.extractedMetadata`.
* **Failure Condition:** If `discovered` is `false`, you must read the `status.error` field, review the server logs, correct the Go implementation in the reconciler, and repeat the compilation and testing sequence until this step passes.
